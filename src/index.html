mvc.ts

I split this project into two versions. A short description of each follows, including a link to their respective working demos.

Version 1

View demo

In version 1, the App class is the Controller in the truest sense, as it coordinates all communication between components. The View calls all registered event listeners when a user interacts with user interface. The Controller listens for View events and acts accordingly when and event occurs - namely, updating the Model, persisting data and messaging the View to update.

Version 2

View demo

In version 2, the Controller initializes the application and manages data persistance. The View handles all user interaction and Model updates. The Model emits events when it's data changes, which triggers the Controller to persist data and message the view to refresh.
